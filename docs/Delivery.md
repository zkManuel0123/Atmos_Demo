# Delivery Notes

最后更新：2026-03-29 16:54:15

## 任务理解

根据 [Introduction.md](/D:/Interview/deepwisdom/docs/Introduction.md) 的要求，这次笔试的重点不是复刻完整 Atoms，而是在有限时间内交付一个：

- 可运行
- 有真实交互
- 有数据持久化
- 有明显 Agent 驱动流程
- 有可视化结果展示
- 可以继续迭代扩展

因此当前 Demo 的范围收敛为一个 `AI Web Builder / Dashboard Builder`：

1. 用户输入自然语言需求
2. 系统调用大模型执行完整 agent tool loop
3. Agent 的规划、工具调用、执行结果在左侧会话流里实时可视化
4. 右侧实时承接最终生成的网页结果
5. 用户可继续 refinement，并保留版本历史

## 当前版本概况

当前完成度可以定义为：

`MVP 已跑通，核心流程已具备真实 Agent 执行、结果预览、版本记忆与可演示的产品外壳。`

当前项目已经不再是最初的静态 schema demo，而是：

- 首页输入需求
- 点击后直接进入主工作区
- 左侧以聊天流形式展示 Agent 执行可视化
- 右侧展示真实网页预览
- 每次生成形成一个版本，可继续迭代

## 已完成内容

### 1. 首页与产品入口

- 已实现独立首页输入区
- 支持中文示例 prompt
- 支持从首页直接发起新任务
- 浏览器重新打开时默认先回到首页，不会被历史状态强制跳进错误页面

### 2. Kimi K2.5 接入

- 已接入 `Kimi K2.5`
- 当前优先级为：
  - `KIMI_API_KEY`
  - `OPENAI_API_KEY`
  - mock fallback
- 已处理 Kimi 在多轮 tool calling 下的兼容问题
- 已处理 Kimi 返回 fenced JSON 时的解析问题

### 3. 完整 Agent Tool Loop

当前后端已不是单次“生成 spec 再返回”的简单接口，而是基于 `/api/generate/stream` 的流式执行链路。

已实现：

- 流式 SSE 事件返回
- 多轮 `model -> tool call -> tool result -> next round`
- 工具执行事件持续推送到前端
- 前端实时消费事件流并渲染到会话区

当前内置工具包括：

- `todo_write`
- `set_build_mode`
- `capture_brief`
- `define_dashboard_schema`
- `define_information_architecture`
- `log_agent_decision`
- `finalize_spec`
- `build_artifact`

### 4. Agent 执行可视化

左侧已改成聊天线程式展示，而不是卡片式工作台。

当前可视化内容包括：

- 用户输入
- 执行计划
- tool call
- tool result
- Agent 输出
- 最终回答

也就是说，用户已经可以看到：

- 当前在做哪一步
- 工具调用了什么
- 执行结果是什么
- 最终产出了什么页面

### 5. 真实网页结果预览

- 右侧预览区通过 `iframe srcDoc` 渲染真实网页
- 不再只是静态说明或 schema 展示
- 已去掉无关代码面板，右侧聚焦结果本身
- 当前工作区布局改成更接近 Adorable / Lovable / Atoms 的 `chat + preview` 结构

### 6. 版本与迭代

- 每次生成保存一个版本
- 支持在当前版本基础上继续 refinement
- 支持切换历史版本查看结果

### 7. 数据持久化

当前使用浏览器本地持久化：

- 项目列表
- 当前激活项目
- 版本历史
- 生成 artifact

这已经满足题目里“具备数据持久化”的基本要求。

### 8. 场景专项生成

已针对 `dashboard / 数据大盘 / 销售看板 / revenue / pipeline` 这类 prompt 做专项模式优化。

例如：

- “帮我制作面向销售的业务数据大盘页面”

会优先按 dashboard 模式生成，而不是普通营销页。

## 当前 UI 状态

本轮 UI 已从最早的轻量 dashboard/workbench 风格，重构为更接近市面上主流 Agent 产品的界面结构。

当前结构是：

- 左侧：会话线程 + Agent 执行过程 + 输入框
- 右侧：浏览器式预览头 + 结果画布

这一版 UI 主要参考了本地对照的 [Adorable](/D:/Interview/deepwisdom/Adorable/Adorable) 工作区结构，但没有直接迁入其完整工程，只抽取了：

- 统一顶部控制栏
- 左会话右预览
- 轻量浏览器壳
- 强化线程感、弱化后台面板感

另外，当前页面上的主要交互和展示文案都已切换为中文。

## 关键取舍

为了保证 6-8 小时内能交出完整可演示的原型，当前主动没有做这些内容：

- 真正的任意应用生成平台
- 复杂代码沙箱
- 用户登录
- 云端数据库
- 一键发布公网链接
- 完整终端/文件系统工具链

当前策略是优先完成一个高完成度的主流程：

- 能跑
- 能看
- 能解释
- 有 Agent 感
- 有产品感

## 当前不足

当前还没有完全做完的部分：

- 在线部署链接还未最终交付
- README 和最终提交说明还需要再统一一版
